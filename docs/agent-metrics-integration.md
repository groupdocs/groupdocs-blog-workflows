# Agent Metrics Integration Guide

This guide explains how agent workflows in this repository report run metrics to the centralized metrics endpoint.

## Overview

Each agent workflow reports a JSON payload at the end of every run via `AGENT_METRICS_ENDPOINT`. The metrics step runs with `if: always()` so it fires on both success and failure. If the endpoint is not configured, the step silently skips without failing the workflow.

### Currently integrated workflows

| Workflow | agent_name | job_type | Items source |
|---|---|---|---|
| `scan-missing-translations.yml` | Docs Translator | Docs Translation | `translations_scan_report.json` |

## Payload schema

```json
{
  "timestamp": "2026-03-29T10:12:45.123456+00:00",
  "agent_name": "Docs Translator",
  "agent_owner": "Vladimir Litvinchik",
  "job_type": "Docs Translation",
  "run_id": "translate_posts_12345678",
  "status": "success",
  "product": "GroupDocs.Total",
  "platform": "All",
  "website": "groupdocs.com",
  "website_section": "Blog",
  "item_name": "Translation",
  "items_discovered": 21,
  "items_failed": 0,
  "items_succeeded": 21,
  "run_duration_ms": 142880,
  "token_usage": 48500,
  "api_calls_count": 44
}
```

### Field reference

| Field | Required | Description |
|---|---|---|
| `timestamp` | auto | ISO 8601 with timezone, generated by `report_metrics.py` |
| `agent_name` | yes | One of: `Docs Translator`, `Content Enhancer`, `KB Article Writer`, `Product Pages Agent`, `Examples Creator`, `Code Generator`, `Keyword Analyzer` |
| `agent_owner` | yes | Default: `Vladimir Litvinchik` |
| `job_type` | yes | E.g. `Docs Translation`, `Content Enhancement`, `KB Article Generation` (use `test` for test entries) |
| `run_id` | auto | `{workflow_slug}_{github.run_id}` or auto-generated UUID |
| `status` | yes | `success` or `failure` |
| `product` | yes | Official product name: `GroupDocs.Total`, `GroupDocs.Viewer`, etc. Use `.Total` if not product-specific |
| `platform` | yes | `.NET`, `Java`, `Python`, `Node.js`, `C++`, or `All` |
| `website` | yes | `groupdocs.com`, `aspose.com`, etc. |
| `website_section` | yes | `Blog`, `Docs`, `KB`, `Product Pages`, `Examples`, etc. |
| `item_name` | yes | `Translation`, `Pages`, `KB Articles`, `keywords`, etc. |
| `items_discovered` | yes | Total items the agent attempted to process |
| `items_failed` | yes | Items that failed |
| `items_succeeded` | yes | Items processed successfully |
| `run_duration_ms` | yes | Wall-clock duration of the workflow in milliseconds |
| `token_usage` | yes | Total LLM tokens consumed (input + output). `0` if no LLM calls |
| `api_calls_count` | yes | Number of LLM API calls made. `0` if no LLM calls |

## How to add metrics to a new workflow

### Step 1 - Record start time

Add this as the first step after checkout:

```yaml
      - name: Record start time
        id: start_time
        run: echo "start_ms=$(date +%s%3N)" >> $GITHUB_OUTPUT
```

### Step 2 - (If using LLM) Add token tracking to your Python script

Add the metrics accumulator near the top of your script:

```python
import json

# Metrics tracking for token usage and API calls
_metrics = {"token_usage": 0, "api_calls_count": 0}


def _track_usage(response):
    """Extract and accumulate token usage from an API response."""
    _metrics["api_calls_count"] += 1
    if hasattr(response, 'usage') and response.usage:
        _metrics["token_usage"] += getattr(response.usage, 'total_tokens', 0)
```

Call `_track_usage(response)` after every `client.chat.completions.create()` call:

```python
response = client.chat.completions.create(model=model, messages=messages)
_track_usage(response)
content = response.choices[0].message.content.strip()
```

Write the metrics file at the end of your script's `main()`:

```python
try:
    with open('my_agent_metrics.json', 'w') as f:
        json.dump(_metrics, f)
except Exception:
    pass  # Don't fail the run over metrics
```

### Step 3 - Add the Report metrics step

Add this as the **last step** in the workflow job:

```yaml
      - name: Report metrics
        if: always()
        env:
          AGENT_METRICS_ENDPOINT: ${{ secrets.AGENT_METRICS_ENDPOINT }}
        run: |
          set +e
          END_MS=$(date +%s%3N)
          START_MS=${{ steps.start_time.outputs.start_ms }}
          if [ -n "$START_MS" ]; then
            DURATION_MS=$((END_MS - START_MS))
          else
            DURATION_MS=0
          fi

          # --- Determine status and item counts ---
          # Adapt this section to your workflow's outputs/files.
          # Example: read from a JSON report your script produced.
          STATUS="success"
          DISCOVERED=0
          SUCCEEDED=0
          FAILED=0

          if [ -f my_report.json ]; then
            DISCOVERED=$(python -c "import json; print(json.load(open('my_report.json'))['total'])" 2>/dev/null || echo "0")
            SUCCEEDED=$DISCOVERED
          else
            STATUS="failure"
          fi

          # --- Read token metrics (skip if no LLM) ---
          TOKEN_USAGE=0
          API_CALLS=0
          if [ -f my_agent_metrics.json ]; then
            TOKEN_USAGE=$(python -c "import json; print(json.load(open('my_agent_metrics.json')).get('token_usage', 0))" 2>/dev/null || echo "0")
            API_CALLS=$(python -c "import json; print(json.load(open('my_agent_metrics.json')).get('api_calls_count', 0))" 2>/dev/null || echo "0")
          fi

          # --- Send metrics ---
          python tools/agent-metrics-reporter/report_metrics.py \
            --agent-name "My Agent Name" \
            --job-type "My Job Type" \
            --status "$STATUS" \
            --run-id "my_agent_${{ github.run_id }}" \
            --product "GroupDocs.Total" \
            --platform "All" \
            --website "groupdocs.com" \
            --website-section "Blog" \
            --item-name "Pages" \
            --items-discovered "$DISCOVERED" \
            --items-succeeded "$SUCCEEDED" \
            --items-failed "$FAILED" \
            --run-duration-ms "$DURATION_MS" \
            --token-usage "$TOKEN_USAGE" \
            --api-calls-count "$API_CALLS" || true
```

Key points:
- `if: always()` ensures metrics fire even on failure
- `set +e` prevents the metrics step itself from failing the job
- `|| true` at the end is a safety net if the reporter script errors
- Extract item counts from whatever JSON/output your agent already produces

```

Required:
  --agent-name TEXT        Agent name
  --job-type TEXT          Job type
  --status {success,failure}

Optional (with defaults):
  --agent-owner TEXT       Default: "Vladimir Litvinchik"
  --run-id TEXT            Auto-generated if omitted
  --product TEXT           Default: "GroupDocs.Total"
  --platform TEXT          Default: "All"
  --website TEXT           Default: "groupdocs.com"
  --website-section TEXT   Default: "Blog"
  --item-name TEXT         Default: "Translation"
  --items-discovered INT   Default: 0
  --items-failed INT       Default: 0
  --items-succeeded INT    Default: 0
  --run-duration-ms INT    Default: 0
  --token-usage INT        Default: 0
  --api-calls-count INT    Default: 0
```

The script uses only Python stdlib (`urllib.request`) - no `pip install` needed.

## Token tracking details

Token capture relies on the OpenAI SDK's `response.usage` object, which the Professionalize API also returns. The `_track_usage` helper safely handles cases where `usage` is `None` or the `total_tokens` attribute is missing.

### Currently tracked scripts

| Script | Metrics file | Call sites |
|---|---|---|
| `tools/blog-post-translator/translate_posts.py` | `translation_metrics.json` (in CWD) | `translate_text()`, `translate_front_matter()`, `review_translation()` |
| `tools/public-release-post-draft/create_draft.py` | `output/draft_metrics.json` | `generate_draft_with_llm()`, `review_full_post_with_llm()`, `refine_front_matter_with_llm()` |

### Adding token tracking to a new script

1. Copy the `_metrics` dict and `_track_usage()` function from either script above
2. Add `_track_usage(response)` after every `client.chat.completions.create()` call
3. Write `json.dump(_metrics, f)` at the end of `main()`
4. Reference the output file path in your workflow's Report metrics step

## Testing

To test metrics reporting without affecting production data, set `job_type` to `test`:

```bash
AGENT_METRICS_ENDPOINT="https://your-endpoint/api/agent-runs" \
python tools/agent-metrics-reporter/report_metrics.py \
  --agent-name "Docs Translator" \
  --job-type "test" \
  --status "success" \
  --items-discovered 5 \
  --items-succeeded 5
```

Test entries with `job_type: test` are auto-cleaned daily between 8-9 PM.
